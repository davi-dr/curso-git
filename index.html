Comandos Git

  1. git init : Inicia um novo repositório vazio no GIT local na pasta atual.
  2. git status : Verifica o status atual do repositório, incluindo arquivos modificados e arquivos na (Staging Area).
  3. git commit -m : Grava as alterações feitas na área de stage como uma nova versão do código.

    4. OBS: Mensagem de Commit: É comum usar -m "Mensagem do commit" junto com git commit -a para adicionar uma mensagem de commit, como em git commit -a -m "Atualizações de arquivos rastreados".

  4. git add + NomeArquivo : Adiciona arquivos selecionados para a área de stage (pronto para ser commitado).
  5. git add . : Adicionar todos os arquivos.
  6. git log : Ver o histórico de commits.
  7. git checkout: Usado para alternar entre BRANCHS, cria novos branches, ou mudar para uma versão anterior do codigo em um branch existente.
  8. git checkout -b nomeBranch: Cria uma nova branch e ja navega para ela.
  9. git branch: Lista as branches do projeto.
  10. git merge nomeBranchDesejada: Permite unir o conteúdo de duas ou mais branches em uma única branch / É usado para integrar alterações feitas em uma branch secundária (Geralmente uma branch de desenvolvimento) na branch Principal (geralmente na branch master/main)
  11. git remote add origin: Usado para conectar o projeto local com o repositório do GitHub.
  12. git push: Para enviar o projeto para o GitHub.
  13. git clone: Puxar o repositório do GitHub e baixar na maquina do usuário / você seleciona a URL do projeto no GitHub que quer baixar.
